## 📊 实施内容

### P0：统一 Key 格式

**修改前：**
```typescript
const cacheKey = `ctx:${agentId}:${hash}`;
```

**修改后：**
```typescript
const memType = 'episodic';  // 或 'semantic'
const cacheKey = `cache:${agentId}:${memType}:${hash}`;
```

**优势：**
- 支持不同类型独立缓存
- 更容易管理和清理

---

### P1：清理重复缓存

**删除：**
- ❌ `ctx:*` (10 个 keys)
- ❌ `search:*` (8 个 keys)

**统一为：**
- ✅ `cache:{agent}:{type}:{hash}`

**优势：**
- 减少 Redis keys 数量
- 避免重复存储
- 简化管理

---

### P2：缓存失效联动

**实施：**
```typescript
// 保存新记忆后
if (redisClient) {
  const pattern = `cache:${agentId}:episodic:*`;
  const keys = await redisClient.keys(pattern);
  if (keys && keys.length > 0) {
    await redisClient.del(...keys);
    debugLog(`Cache invalidated: ${keys.length} keys`);
  }
}
```

**优势：**
- 避免脏数据
- 保证数据一致性

## 📋 新缓存策略

### Key 格式

```
cache:{agent_id}:{mem_type}:{content_hash}
```

**示例：**
```
cache:main:episodic:1234567890
cache:alisa:semantic:9876543210
```

---

### TTL 策略

| 记忆类型 | TTL | 说明 |
|---------|-----|------|
| **episodic** | 1800s (30 分钟) | 对话历史 |
| **semantic** | 7200s (2 小时) | 知识记忆 |
| **procedure** | 7200s (2 小时) | 流程记忆 |

---

### 失效策略

| 触发条件 | 失效范围 |
|---------|---------|
| **保存新记忆** | `cache:{agent}:episodic:*` |
| **更新记忆** | `cache:{agent}:*:{hash}` |
| **删除记忆** | `cache:{agent}:*:{hash}` |
| **TTL 过期** | 自动删除 |
