## 🏗️ v6.0 新架构

### 核心原则

**evolution-v5 的职责：**
1. **数据存储** - LanceDB 持久化
2. **缓存层** - Redis L0-L3 缓存
3. **配置管理** - LanceDB 共享配置
4. **工具提供** - 供 LLM 调用
5. **钩子注入** - before_prompt_build 注入上下文

**LLM 的职责：**
1. **推理分析** - think 功能
2. **摘要生成** - learn 功能
3. **情感分析** - feel 功能
4. **任务规划** - execute 功能
5. **优先级判断** - perceive 功能

## 📋 模块变更

### think（推理模块）

**v5.7：**
```python
# 自己实现推理逻辑
def extract_factors(content): ...
def generate_conclusion(content): ...
```

**v6.0：**
```python
# 仅返回元数据，推理交给 LLM
async def think(content, cache, model, agent_id):
    return {
        'status': 'ok',
        'metadata': {
            'handled_by': 'llm'  # 由 LLM 处理
        }
    }
```

**代码量：** 100 行 → 40 行 (-60%)

## 💡 核心洞察

### 架构设计原则

1. **单一职责** - 每个组件只做一件事
   - LLM：推理/分析/生成
   - evolution-v5：存储/缓存/配置

2. **不要重复造轮子** - 使用框架已有能力
   - OpenClaw 已有 LLM → 不用自己实现推理
   - evolution-v5 提供存储 → 专注数据管理

3. **工具化思维** - 提供工具，让 LLM 调用
   - 不是"我实现功能"
   - 而是"我提供工具，LLM 决定何时使用"
